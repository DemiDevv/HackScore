# HackScore — Инструкция для Claude Code

> Этот пакет содержит **полный дизайн-прототип** веб-приложения HackScore и **план пошаговой разработки**. Всё готово к реализации.

## Что здесь есть

```
handoff/
├── CLAUDE.md                ← ЭТА ИНСТРУКЦИЯ (читай первым)
├── DESIGN_TOKENS.md         ← Все цвета, шрифты, размеры, тени, анимации
├── SCREEN_MAP.md            ← Карта всех экранов + маппинг компонентов на shadcn/ui
├── PLAN_CLAUDE_CODE.md      ← Пошаговый план разработки (16 шагов)
├── DESIGN_BRIEF.md          ← Оригинальный дизайн-бриф
└── prototype/
    ├── HackScore.html       ← Рабочий интерактивный прототип (открой в браузере!)
    └── hs-styles.css        ← CSS-токены и анимации
```

## Как использовать

### 1. Открой прототип
Открой `prototype/HackScore.html` в браузере. Это **полностью кликабельный прототип** со всеми 15+ экранами:
- **Авторизация**: Login + Register (с выбором роли)
- **Участник**: Хакатоны, Команда, Загрузка решения (stepper + 4 артефакта + результаты), Алго-задачи (с редактором кода), Результаты
- **Жюри**: Список решений (фильтры), Детальная оценка (табы + sticky панель слайдеров)
- **Организатор**: Управление хакатоном (статусы), Критерии (веса), Команды, Алго-задачи, Лидерборд (подиум)

**Демо-аккаунты** (предзаполнены на форме логина):
- `team1@hackscore.ru` → Участник
- `jury1@hackscore.ru` → Жюри
- `admin@hackscore.ru` → Организатор

Переключение ролей: клик по аватару в header → "Переключить роль"

### 2. Изучи дизайн-токены
Файл `DESIGN_TOKENS.md` — полный справочник по визуалу:
- Цвета (фоны, акценты, статусы, текст)
- Типографика (шрифты, размеры, веса)
- Радиусы, отступы, тени
- Спецификации каждого UI-компонента (Button, Badge, Card, Input, Modal, Tabs, etc.)
- Градиенты и backdrop-blur эффекты
- Анимации и transitions

### 3. Изучи карту экранов
Файл `SCREEN_MAP.md`:
- Навигация по ролям (sidebar структура)
- Детальное описание каждого экрана и его элементов
- **Маппинг на shadcn/ui** — какой наш компонент чему соответствует
- Tailwind config для цветовой палитры

### 4. Следуй плану разработки
Файл `PLAN_CLAUDE_CODE.md` — **16 шагов**, каждый с готовым промптом:
- **Шаг 0**: Инициализация монорепо + Docker Compose
- **Шаг 1**: Модели БД + миграции Alembic
- **Шаг 2**: JWT-аутентификация
- **Шаг 3**: CRUD хакатонов, команд, критериев
- **Шаг 4**: Загрузка артефактов (file upload)
- **Шаги 5-8**: Celery workers (код, документация, презентация, скринкаст)
- **Шаг 9**: Система оценки + панель жюри
- **Шаг 10**: Алгоритмический модуль (sandbox runner)
- **Шаг 11**: Seed-данные
- **Шаги 12-15**: Frontend (Layout+Auth → Участник → Жюри → Организатор)
- **Шаг 16**: Финальная интеграция

| Контейнеры | Docker Compose |
| Auth | JWT + bcrypt |

## Ключевые принципы дизайна

1. **Тёмная тема** — основа. Фон #0A0A0F, карточки #12121A
2. **Glass-morphism** — header с backdrop-blur
3. **Градиенты** — акцентные кнопки, лого (индиго → бирюзовый)
4. **Mesh-gradient** — декоративный фон на страницах авторизации
5. **Subtle glow** — box-shadow с цветом акцента для важных элементов
6. **Lucide icons** — тонкие, stroke-based, 20px default
7. **Space Grotesk** для заголовков + **DM Sans** для текста + **JetBrains Mono** для кода
8. **Анимации**: fadeIn для контента, scaleIn для модалок, slideR для тостов, pulse для статусов

## Важные детали

- **Все ID в БД — UUID v4**
- **Все FK с ondelete CASCADE**
- **Docker Compose**: `docker compose up --build` должен поднимать всё с нуля
- **Seed-данные**: автоматически при первом запуске
- **API-документация**: Swagger на `/api/docs`
- **Формула итогового балла**: `Σ(criterion_score × weight)`, weight в долях (сумма = 1.0)
- **Авто-критерии**: score из check_results. **Эксперт-критерии**: среднее по всем jury scores
